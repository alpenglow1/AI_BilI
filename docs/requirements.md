# AI 记账 App 需求文档

> 版本：v1.0
> 日期：2026-06-15

---

## 一、项目概述

一款基于 AI 自然语言识别的智能记账 App。用户通过与 AI 聊天的方式完成记账，支持账单明细查看、编辑及个人信息管理。

---

## 二、技术栈

| 层级 | 技术选型 |
|------|---------|
| 前端 | React Native (Expo) |
| 后端 | Express + TypeScript |
| 数据库 | Supabase PostgreSQL |
| ORM | Drizzle |
| AI 模型 | 智谱 GLM-4.7 |
| 认证 | Supabase Auth |

---

#### AI 回复两种模式

| 场景 | 用户输入示例 | AI 回复形式 |
|------|------------|------------|
| ✅ 识别为账单 | "打车花了23块" | 显示**账单预览卡片** |
| 💬 无法识别 / 闲聊 / 查询 | "Hi" / "这月花了多少" | 显示**普通文字气泡** |

#### 账单预览卡片规范
```
┌──────────────────────────┐
│  🚗  打车去火车站    -23.00
│                  │
└──────────────────────────┘
  [✅ 确认入账]  [✏️ 修改]
```
- 卡片内：二级分类 + 账单标题 + 金额（不含日期）
- 卡片外下方：操作按钮

- 点击「修改」→ 弹出编辑 Modal，调整后再确认

#### 聊天历史持久化
- 对话记录保存到数据库
- 每次进入页面自动加载历史消息

#### 清除显示功能
- 顶部「清除显示 🗑️」按钮
- 弹出确认 Dialog：「确认清除聊天显示？数据不会被删除」
- 确认后：本地记录 `clearedAt` 时间戳，界面只显示该时间之后的消息
- **数据库中的聊天记录永久保留，不删除**

## 六、数据库设计

### categories 二级分类表
```sql
categories
├── id          uuid, PRIMARY KEY
├── type        'income' | 'expense'    -- 归属的一级分类
├── name        varchar(50)             -- 二级分类名称（如：餐饮、交通、工资）
├── is_default  boolean, DEFAULT true   -- 是否为系统预设
├── user_id     uuid, NULLABLE          -- 创建者ID（预设分类为 null，自定义分类关联用户）
├── sort_order  integer, DEFAULT 0      -- 排序权重
├── created_at  timestamp
└── updated_at  timestamp
```

### bills 账单表
```sql
bills
├── id          uuid, PRIMARY KEY
├── user_id     uuid, FK → auth.users
├── type        'income' | 'expense'    -- 冗余一级分类，方便快速统计查询
├── category_id uuid, FK → categories.id-- 关联具体的二级分类
├── name        varchar(100)            -- 账单标题（具体备注，如：麦当劳）
├── amount      decimal(10, 2)          -- 金额
├── date        date                    -- 账单时间（精确到年月日）
├── created_at  timestamp
└── updated_at  timestamp
```

### chat_messages 聊天记录表
```sql
chat_messages
├── id          uuid, PRIMARY KEY
├── user_id     uuid, FK → auth.users
├── role        'user' | 'assistant'    -- 消息来源
├── content     text                    -- 消息内容
└── created_at  timestamp
```

> `users` 表由 Supabase Auth 自动维护，无需手动创建

## 七、API 接口清单

### 认证相关
| 方法 | 路径 | 功能 |
|------|------|---------|
| POST | `/auth/register` | 用户注册 |
| POST | `/auth/login` | 用户登录，返回 Token |
| POST | `/auth/change-username` | 修改用户名 |
| POST | `/auth/change-password` | 修改密码 |
| POST | `/auth/logout` | 退出登录 |

### 分类相关
| 方法 | 路径 | 功能 |
|------|------|---------|
| GET | `/categories?type=expense` | 获取指定类型的二级分类列表（支出/收入） |

### 账单相关
| 方法 | 路径 | 功能 |
|------|------|---------|
| GET | `/bills?date=2026-06-01` | 获取指定日期账单列表（用于首页展示） |
| POST | `/bills` | 新增账单（AI 确认入账 或 手动添加） |
| PUT | `/bills/:id` | 编辑账单 |
| DELETE | `/bills/:id` | 删除账单 |

### AI 对话相关
| 方法 | 路径 | 功能 |
|------|------|---------|
| GET | `/chat/messages` | 获取历史聊天记录 |
| POST | `/chat/send` | 发送消息，后端统一处理（录入/查询/闲聊） |

> **注**：「清除聊天显示」功能的 `clearedAt` 时间戳仅存储在**本地设备**（AsyncStorage），不涉及后端接口，数据库中的聊天记录永久保留。

## 八、AI 对话流程

### AI 识别的三种模式

| 模式 | 用户输入示例 | 后端处理 | 前端渲染 |
| :--- | :--- | :--- | :--- |
| ✅ 账单录入 | "打车花了23块" | AI识别→返回结构化JSON | 账单预览卡片 |
| 📊 数据查询 | "这个月餐饮花了多少" | AI识别查询意图→后端查DB→结果传AI→自然语言回复 | 普通文字气泡 |
| 💬 闲聊/其他 | "你好" / "帮我分析消费习惯" | AI直接回复 | 普通文字气泡 |

      ↓
  ┌─────────────────────────────────────────────┐
  │  意图判断结果：                               │
  │                                              │
  │  ① 账单录入 → 返回结构化 JSON               │
  │  {                                           │
  │    "intent": "bill",                         │
  │    "type": "expense",                        │
  │    "category": "餐饮",                        │
  │    "name": "吃饭",                            │
  │    "amount": 100,                            │
  │    "date": "2026-06-14"  ← AI解析相对日期     │
  │  }                                           │
  │                                              │
  │  ② 数据查询 → 返回查询参数 JSON              │
  │  {                                           │
  │    "intent": "query",                        │
  │    "startDate": "2026-06-01",                │
  │    "endDate": "2026-06-30",                  │
  │    "category": "餐饮"                        │
  │  }                                           │
  │                                              │
  │  ③ 闲聊/无法识别 → 直接返回文字回复          │
  │  {                                           │
  │    "intent": "chat",                         │
  │    "reply": "你好！..."                       │
  │  }                                           │
  └─────────────────────────────────────────────┘
      ↓
  【仅 intent = "query" 时】
  后端用查询参数检索 bills 数据库
  将查询结果拼入 Prompt，第二次调用 GLM-4.7
  获取自然语言形式的数据摘要回复
      ↓
前端根据 intent 字段决定渲染方式：
  - "bill"  → 账单预览卡片（含确认入账/修改按钮）
  - "query" / "chat" → 普通文字气泡

【date 字段的日期解析规则】
  System Prompt 中注入今日日期，AI 负责将相对时间解析为绝对日期：
  - "今天" / 未提及日期  → 今日日期
  - "昨天"              → 今日 - 1 天
  - "上周五"            → 最近一个周五的日期
  - "6月1日"            → 当年对应日期

【确认入账的数据流】
  用户点击「确认入账」→ 前端将 bill JSON 的完整字段
  （type, category_id, name, amount, date）发送至 POST /bills
  → 后端写入数据库
```

## 十、建议开发顺序

```
阶段 1：基础搭建
  ✦ 后端项目初始化（Express + TS + Drizzle）
  ✦ 数据库 Schema 定义 + Migration（含 categories、bills、chat_messages 三张表）
  ✦ categories 预置数据 Seed（10条支出分类 + 6条收入分类，Migration 后立刻执行）
  ✦ Supabase Auth 接入（登录 / 注册 / 修改用户名 / 修改密码 / 退出登录接口）

阶段 2：账单核心功能
  ✦ 分类接口 GET /categories（后端，前端账单编辑弹窗的依赖项）
  ✦ 账单 CRUD 接口（后端，POST /bills 需接收 category_id 与 date 字段）
  ✦ 首页账单列表 + 日期导航（前端）
  ✦ 账单卡片编辑弹窗（含二级分类网格选择器 CategoryGridPicker、联动切换逻辑）
  ✦ 删除账单（含 ConfirmDialog 二次确认）

阶段 3：AI 对话功能
  ✦ 聊天消息存取接口（后端，GET /chat/messages / POST /chat/send）
  ✦ GLM-4.7 接入 + System Prompt 设计
     - 注入今日日期、预设二级分类列表、三种意图判断规则
     - 处理三条链路：
       · bill  → 解析结构化JSON（含 date 日期解析），返回账单预览卡片
       · query → 提取时间/分类参数，查询DB，二次调用GLM生成自然语言摘要
       · chat  → 直接返回文字回复
  ✦ 前端聊天界面 + 账单预览卡片交互（确认入账 / 修改/取消）
  ✦ 清除对话显示逻辑（clearedAt 仅存本地 AsyncStorage，不影响数据库）

阶段 4：个人信息页 & 收尾
  ✦ 「我」页面开发（用户名展示、邮箱展示）
  ✦ 修改用户名功能（ChangeUsernameModal + /auth/change-username 接口联调）
  ✦ 修改密码功能（ChangePasswordModal + /auth/change-password 接口联调）
  ✦ 退出登录（清除本地 Token + ConfirmDialog 确认，跳回登录页）
  ✦ 整体联调测试 + UI 打磨
